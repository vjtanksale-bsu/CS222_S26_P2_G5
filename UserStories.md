# User Stories

## User Story 01

As a student, I want to be able to **see all course numbers** that are offered so that I can see all of my options in one place.

## Acceptance Criteria
1. Main menu has an option to view all available courses and their information. 
2. While creating a schedule, the course numbers are available to see on screen.
3. All course numbers available are presented clearly on the screen


## User Story 02

As a student I should be able to **enter the amount of courses** I would like to register for so I can keep track of how many class I am in.

 ## Acceptance Criteria

1. The main menu should give an option to create a schedule.
2. The program should prompt the user to enter how many courses they want to register for
3. If the input from the user is not an integer, the program should produce an error message.
4. An empty numbered list is created and visual to the user
5. The user can add or remove courses from the list


## User story 03

As a student I should be able to **enter distinct course numbers** so I can create my schedule.

## Acceptance Criteria
1. There is a prompt for the user to enter the course number for a class
2. The program adds the course number and it's information to the user's schedule
3. The schedule list is updated for the user to see.
4. The program should prevent a user from entering the same course number multiple times.
5. If no course matches the entered number, then the program must produce an error message.


## User Story 04

As a student I should be able to **see my schedule** and class information and be informed if I do not have a schedule so that I know when and where each class is.

## Acceptance Criteria
1. The main menu should have the option to view schedule.
2. When selected, the program should show the list including course number, section number, meeting times and days.
3. If no list exist, there should be a message informing the user of such and asking if they would like to create one.
